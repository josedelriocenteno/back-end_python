# Git (Local) vs. GitHub (Remoto)

Es el error más común del principiante: pensar que Git y GitHub son lo mismo. Son herramientas complementarias pero muy diferentes.

## 1. Git: El Motor
Git es el programa que instalas en tu ordenador. Es el que gestiona las ramas, los commits y el historial localmente.
- **Vive en:** Tu carpeta `.git` local.
- **Funciona:** Sin internet.
- **Propiedad:** Es software libre (Open Source).

## 2. GitHub: El Almacén y Red Social
GitHub (o GitLab/Bitbucket) es una plataforma en la nube que aloja repositorios de Git.
- **Propósito:** Compartir código con otros, gestionar proyectos y centralizar el flujo de trabajo.
- **Funciones extra:** Pull Requests, Issues, Actions, Wikis, Visualización de código.

## 3. La analogía del mundo real
- **Git = Tu Procesador de Textos (Word/Google Docs):** Donde escribes y controlas las versiones de tu documento privado.
- **GitHub = Dropbox / Google Drive:** Donde subes el documento para que otros lo vean, lo editen o colaboren.

## 4. ¿Puedo usar Git sin GitHub?
**Sí.** Puedes tener todo un proyecto versionado en tu PC y nunca subirlo a ningún sitio.

## 5. ¿Puedo usar GitHub sin Git?
**A duras penas.** GitHub permite subir archivos a través de la web, pero pierdes el 99% del poder y la velocidad que te da el CLI de Git.

## Resumen: Un equipo inseparable
Git te da el control técnico. GitHub te da la colaboración social. La mayoría de proyectos profesionales utilizan Git para trabajar y GitHub para integrar y desplegar.
