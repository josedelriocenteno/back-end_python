# ¿Qué es Data Engineering? El fontanero del dato

Mientras que los científicos de datos se centran en obtener valor y predicciones, los **Ingenieros de Datos** se centran en asegurar que los datos estén disponibles, sean fiables y lleguen a tiempo.

## 1. La analogía de la fontanería
Imagina una ciudad (una empresa) que necesita agua (datos). 
- El **Data Scientist** es el chef que usa el agua para cocinar platos increíbles.
- El **Data Engineer** es quien diseña las tuberías, asegura el filtrado (limpieza), gestiona la presión (caudal) y construye los embalses (Data Warehouse) para que al abrir el grifo siempre salga agua limpia.

## 2. Los tres pilares
1. **Disponibilidad:** Que el dato esté donde tiene que estar.
2. **Fiabilidad:** Que el dato sea correcto (no haya duplicados ni nulos inesperados).
3. **Escalabilidad:** Que el sistema funcione igual de bien con 1.000 filas que con 1.000 millones.

## 3. El ciclo de vida: Ingesta -> Transformación -> Carga
Un ingeniero de datos pasa la mayor parte de su tiempo construyendo **Pipelines** que mueven datos desde fuentes diversas (APIs, bases de datos de clientes) hacia sistemas analíticos donde puedan ser consultados de forma masiva.

## 4. Diferencia con DBA (Administrador de Base de Datos)
- El **DBA** se centra en mantener una base de datos específica funcionando (copias de seguridad, tuning).
- El **Data Engineer** se centra en el **flujo del dato** a través de múltiples sistemas.

## Resumen: La base de la pirámide
Sin ingeniería de datos, no hay inteligencia artificial ni analítica. Somos los encargados de construir los cimientos sobre los que se apoya toda la estrategia de datos de una compañía moderna.
