# Issues y Projects: Gestión de tareas profesional

GitHub no es solo un disco duro en la nube; es una herramienta de gestión de proyectos. Como desarrollador senior, pasarás tanto tiempo en el código como gestionando tareas.

## 1. ¿Qué es una Issue?
Es un ticket que representa una tarea, un bug o una idea.
- **Buen Issue:** Describe el problema, pasos para reproducirlo (si es bug) y contexto técnico.
- **Etiquetas (Labels):** Usa colores para clasificar: `bug`, `enhancement`, `urgent`, `documentation`.

## 2. Conectar Código con Tareas
Puedes cerrar una Issue automáticamente desde un commit o una PR usando palabras clave:
`Fixes #123` o `Closes #45`. 
Cuando el código se mergee en `main`, GitHub cerrará la Issue por ti.

## 3. Milestones (Hitos)
Sirven para agrupar Issues en el tiempo. Ej: "Versión 1.0", "Sprint de Marzo". Te permiten ver una barra de progreso de cuánto falta para completar un objetivo.

## 4. GitHub Projects (Tableros Kanban)
Es el "Trello" integrado en GitHub.
- Columnas típicas: `Todo`, `In Progress`, `Review`, `Done`.
- Permite mover las Issues visualmente y tener una visión global de la carga de trabajo del equipo.

## 5. GitHub Discussions
A diferencia de las Issues (que son tareas), Discussions es para debates abiertos, ideas locas o preguntas frecuentes. Mantiene las Issues "limpias" de ruido de conversación.

## Resumen: Organización es Poder
Dominar la gestión de tareas dentro de GitHub te permite ser más autónomo y transparente. Ayuda a que el equipo sepa en qué estás trabajando sin necesidad de reuniones diarias eternas.
