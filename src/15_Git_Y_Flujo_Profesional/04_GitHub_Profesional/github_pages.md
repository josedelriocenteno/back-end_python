# GitHub Pages: Documentación y Web estática gratuita

A menudo necesitas que tu proyecto tenga un sitio web que lo explique o que muestre una demo. GitHub Pages es la solución integrada y gratuita.

## 1. ¿Cómo funciona?
GitHub toma los archivos HTML/CSS/JS de una de tus ramas (normalmente `gh-pages` o una carpeta `/docs` en `main`) y los publica en una URL pública: `tu-usuario.github.io/tu-repo`.

## 2. Casos de Uso
- **Documentación:** Publicar manuales creados con herramientas como Sphinx o MkDocs.
- **Landing Page:** Una web de presentación para tu proyecto.
- **Portfolio:** Tu currículum online alojado directamente desde tu código.

## 3. Activación
1. Ve a **Settings** -> **Pages** en tu repositorio.
2. Elige la rama origen.
3. GitHub te dará la URL a los pocos segundos.

## 4. Dominios Personalizados
Puedes conectar tu propio dominio (ej: `www.miproyecto.com`) de forma sencilla y GitHub te regalará el certificado SSL (HTTPS) automáticamente.

## 5. Jekyll y Generadores de Sitios Estáticos
GitHub Pages tiene soporte nativo para **Jekyll**. Escribes en Markdown y el servidor de GitHub lo convierte en un blog o sitio web completo sin que tengas que configurar nada.

## Resumen: Visibilidad instantánea
Tu código cobra vida cuando otros pueden verlo funcionando o leer su documentación de forma cómoda. GitHub Pages es la forma más profesional y eficiente de dar ese paso final.
